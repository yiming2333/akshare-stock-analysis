**当前版本**: v1.2.0 | **更新日期**: 2026-05-01

## ✨ 项目简介

AKShare Stock Analysis Skill 是一个专为 **A 股市场** 设计的股票智能诊断工具。它整合了**基本面、技术面、资金面和消息面**四大维度的数据，通过单次命令即可输出结构化的 JSON 分析报告。

项目以 **[AKShare](https://github.com/akfamily/akshare)** 为核心数据底座，AKShare 是目前国内最主流的开源金融数据接口库之一，整合了超过 20 家数据源，统一返回 pandas DataFrame 格式。本项目在此基础上进行了封装和增强，专为 AI Agent 和量化研究者打造「开箱即用」的个股诊断能力。

### 🎯 适用场景

- **AI Agent / LLM 工具调用** — 为智能助理提供实时的 A 股分析能力
- **个人量化研究** — 快速获取个股多维度数据，验证策略假设
- **投资辅助决策** — 综合基本面、资金面、技术面信号，辅助判断买卖时机
- **研报与评论生成** — 获取结构化数据，支撑自动化分析报告撰写

### ⚠️ 注意事项

- **学术研究用途**：本项目及其底层依赖的 AKShare 均声明**仅用于学术研究目的**，数据来源于公开财经网站抓取，相关风险由使用者自行承担。
- **数据源稳定性**：AKShare 本质为爬虫类数据源，依赖公开网页接口，存在**接口不稳定或随时失效**的可能，不建议用于实盘高频交易。若追求更高稳定性，建议搭配本地缓存机制使用。
- **数据时效性**：本项目主要输出日频数据，不适合日内高频回测。
- **不构成投资建议**：本工具输出仅供学习与研究参考，不构成任何买卖建议，投资决策需自行判断。

## 🔍 核心功能

### 四维深度诊断引擎

| 维度 | 数据源 | 核心指标 |
|:---:|:---:|:---|
| 📊 **基本面** | 通达信(pytdx) + Baostock | 股本结构、净资产/BVPS、利润表、杜邦分析、PE/PB/PS、PEG、历史分位数、现金流质量（CFO/NP/OR）、同行横向排名 |
| 💰 **资金面** | AKShare（东财数据中心） | 日频主力/超大单/大单/中单/小单净流入、主力持仓占比、北向资金流向、融资融券余额 |
| 📈 **技术面** | Baostock历史K线 + 内置计算 | MA(5/10/20/30/60/120)、MACD、KDJ、RSI、WR、BOLL、ADX、ATR、量价关系、多周期涨跌幅 |
| 📰 **消息面** | ProSearch（腾讯元宝） | 最新公告、研究报告、舆情新闻、政策消息 |

> 完整的 JSON 输出结构参见下文「输出示例」章节。

### v1.2.0 更新亮点（2026-05-01，7项核心修复）

1. **新浪财经 PE/PB 读取补充** — 提升实时估值获取的稳定性

6. **Baostock PE/PB 合理性校验** — 偏差>50%时自动 fallback 至新浪备选源
7. **mktcap 执行顺序修复** — 修复市值与股本之间的计算顺序依赖问题

### 数据流详解

1. **数据获取层（四路并行）**
   - **`pytdx`（通达信协议）**：通过 TDX 协议连接免费行情服务器（默认 `180.153.18.170:7709`），获取实时行情、股本结构、财务指标。该数据源可直接穿透券商底层接口，无需爬取网页。
   - **`Baostock`**：国内开源的证券数据平台，负责历史日 K 线、财务报告（利润表、资产负债表）、PE/PB 历史百分位、杜邦分析拆解。
   - **`AKShare`**：从东方财富数据中心采集资金流向明细（主力/超大单/大单/中单/小单净流入），同时采集合约数据、研究报告列表、限售解禁日历。
   - **`新浪财经`**：作为实时行情的备选数据源（`hq.sinajs.cn`），当 pytdx 连接失败时自动 fallback。

2. **指标计算层**
   - 基于 Baostock 历史 K 线**内置计算** MA、MACD、KDJ、RSI、WR、BOLL、ADX、ATR、量价关系等全量技术指标
   - 采用**双引擎可靠性保障**：当某数据源（如 Baostock PE/PB）出现 >50% 的偏差时，自动切换至新浪备选源
   - PEG 增长率使用 EPS Q1 口径，并设有 **300% 基数保护上限**，防止微利年度 PEG 被异常放大

3. **同行比对层**
   - 根据股票代码映射至 11 个行业 Peer 分组，计算该股在各指标上的**同行业四分位排名及百分位分位数**
   - 实时产出该股票在所属板块的「核心指标横向排序」

4. **输出组装层**
   - 将以上四维数据统一组装为**结构化的 JSON 对象**
   - 搭载 **综合评分 + 三周期预测 + 仓位建议** 等高级研判维度

5. **消息面采集层（可选）**
   - 通过 Node.js 调用 `ProSearch`（腾讯元宝底层爬虫引擎），采集最新 7 日的研报、舆情与公告，不与 Python 主流程耦合。

### 技术栈

```yaml
语言: Python 3.9+
数据接口:
  - akshare (东方财富/同花顺/新浪财经)
  - baostock (免费证券数据平台)

  - requests (新浪备选)

  - pandas (数据处理)
  - numpy (数值计算)
  - json (结果序列化)
AI/LLM (可选):
  - ProSearch / 腾讯元宝 (消息面采集)
```

## 📦 安装与使用

### 环境要求

- Python 3.9 或更高版本（64位）
- pip 包管理器

### 1. 克隆项目

```bash
git clone https://github.com/yiming2333/akshare-stock-analysis.git
cd akshare-stock-analysis
```

### 2. 安装依赖

```bash
pip install baostock pytdx akshare pandas numpy requests
```

> **注意**：以上依赖均为国内金融数据领域的高频使用库，新版各库均已在 PyPI 持续维护，安装时若出现网络超时，可切换国内镜像源（如清华、阿里云）。

### 4. 可选参数

| 参数 | 说明 |
|:---|:---|
| `--no-fund-flow` | 跳过资金流数据采集（加速执行） |
| `--no-news` | 跳过消息面采集 |
| `--output <file>` | 将 JSON 输出保存到指定文件 [9†L6-L7] |


### 5. 完整 Workflow（含消息面）

当需要获取完整信息（含最新舆情消息）时，可执行如下 Node.js 命令（前提是 ProSearch 已部署）：

```bash
# Step1: 四项核心数据（基本面+技术面+资金面）
python scripts/akshare_query.py 600519 --output core_data.json

# Step2: 消息面采集（7日内新闻/研报）
node "/opt/prosearch/prosearch.cjs" --keyword="贵州茅台 最新消息" --freshness=7d --industry=news
```

> 推荐使用 `--output` 参数以便多步联动。

## ⚙️ 配置说明

### 数据源服务器配置（国内专用）

| 源 | 默认 IP | 端口 | 说明 |
|:---|:---|:---|:---|
| 通达信（TDX） | `180.153.18.170` | 7709 | 通用行情服务器，本脚本默认采用该地址 |
| Baostock | 无需配置 | — | 直接调用官方接口 |
| AKShare | 无需配置 | — | 东方财富数据中心，动态采集 |
| 新浪财经备选 | `hq.sinajs.cn` | 80 | 内置容灾 fallback |

> 通达信服务器地址若出现连接失败，可自行更换为其它公共行情主站（搜索“通达信免费行情主站列表”）。

### 缓存机制

本脚本提供 **`.cache/` 目录缓存**，用于临时保存当日各数据源的 Baostock/AKShare Session 和部分财报数据，防止同一启动时间内重复拉取加重服务端负载。缓存文件的超时容忍度约为 10 分钟～1 日，符合非高频分析的批量操作场景。

## 🔧 常见问题（FAQ）

### Q1: 提示 `Missing dependency: ...`

确保已通过 `pip install` 安装全部依赖。若出现 `baostock` / `pytdx` 安装失败，可尝试升级 pip 后再安装：`pip install --upgrade pip`。PyPI 上的这些包都是国内量化领域的常用包，依赖极少，很好拉取。

### Q2: 数据获取失败怎么办？

1. **检查网络连通性**：确保能访问通达信 IP:7709 或新浪财经网站。
2. **检查防火墙设置**：部分企业内网可能限制未知 TCP 端口。
3. **使用备选参数**：若某条数据源持续失败，可运行 `--no-fund-flow` 或 `--no-news` 临时跳过。
4. **刷新缓存**：删掉 `.cache` 目录下的缓存文件再重试。

### Q3: 脚本输出 JSON 不完整？

通常由于某数据源超时（如北向资金拉取失败）导致某 section 内嵌 `"error": ...`。检查控制台的 stderr 输出（非 JSON 部分）定位该数据源的具体报错。

### Q4: 能否支持港股/美股？

当前脚本主要面向 A 股市场，数据源（pytdx/Baostock/东财）均以 A 股为主。若需扩展至港股/美股，可参考项目结构修改数据获取函数。

### Q5: 如何自定义 Peer 分组？

直接编辑 `akshare_query.py` 开头的 `PEER_GROUPS` 字典，增加或修改对应的股票代码列表即可。
